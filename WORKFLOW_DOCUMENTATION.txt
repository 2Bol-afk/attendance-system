================================================================================
                          1. SYSTEM OVERVIEW
================================================================================

The Student Attendance System is a web-based application built with Django that
manages student attendance across multiple courses, subjects, and sections.

KEY COMPONENTS:
├── User Management (Admin, Teacher, Student, Parent)
├── Academic Management (Courses, Subjects, Assignments)
├── Attendance Tracking (Present, Absent, Late)
├── Dashboard Views (Role-specific)
└── Reporting System (Multiple report types)

USER ROLES:
├── Admin: Full system access and management
├── Teacher: Mark attendance, view student lists
├── Student: View personal attendance records
└── Parent: View children's attendance reports

                   6. ATTENDANCE RECORDING WORKFLOW
================================================================================

6.1 ATTENDANCE RECORDING PROCESS
──────────────────────────────────
Step 1: Teacher logs into system
Step 2: Navigate to Mark Attendance page
Step 3: Select Subject Offering
    └── System validates: teacher must be assigned to offering
Step 4: System loads students
    └── Filter: course + year + section matching offering
Step 5: Select Date & Time
    ├── Date: Default = today (can change)
    └── Time: Default = current time (can change)
Step 6: System checks for existing records
    ├── If records exist for date → Pre-fill status
    └── If no records → Show empty form
Step 7: Teacher marks attendance
    ├── For each student:
    │   ├── Select status: Present / Absent / Late
    │   └── System validates selection
    └── Can mark all students or subset
Step 8: Teacher clicks "Save Attendance"
Step 9: System processes attendance
    ├── For each marked student:
    │   ├── Check if record exists (student + offering + date)
    │   ├── If exists → Update record
    │   └── If not exists → Create new record
    └── Save: student, subject_offering, date, time, status
Step 10: System validates unique constraint
    └── Prevents duplicate: (student, subject_offering, date)
Step 11: Show success message
Step 12: Redirect to same page with filters preserved

6.2 ATTENDANCE DATA STRUCTURE
──────────────────────────────
Each Attendance Record Contains:
├── Student (ForeignKey to StudentProfile)
├── Subject Offering (ForeignKey to SubjectOffering)
├── Date (DateField)
├── Time (TimeField)
└── Status (CharField: 'present', 'absent', 'late')

Unique Constraint:
└── (student, subject_offering, date) - One record per student per subject per day

6.3 ATTENDANCE VALIDATION RULES
────────────────────────────────
Rule 1: Teacher can only mark attendance for assigned subjects
Rule 2: Only students enrolled in the subject offering can have attendance
Rule 3: One attendance record per student per subject per day
Rule 4: Date cannot be in the future (system validation)
Rule 5: Status must be one of: Present, Absent, Late

    └── Course must exist

10.2 TEACHER ASSIGNMENT WORKFLOW
─────────────────────────────────
Step 1: Admin navigates to /academic/assignments/add-page/
Step 2: Select Teacher (from datalist)
Step 3: Select Year Level
    └── System filters subjects by year_level
Step 4: Select School Year (e.g., 2024-2025)
Step 5: For each subject:
    ├── Check sections to assign (A, B, C, D)
    └── System checks if offering already exists
Step 6: System creates SubjectOffering:
    ├── Links teacher to subject
    ├── Sets year level
    ├── Sets section
    └── Sets school year
Step 7: System automatically enrolls students:
    └── Students matching (course + year + section) → enrolled in subject
Step 8: Save → Assignment created

10.3 STUDENT ENROLLMENT WORKFLOW
────────────────────────────────
Step 1: When student is created/edited:
    ├── Admin selects subjects
    └── System links student to subjects (ManyToMany)
Step 2: When subject offering is created:
    ├── System finds students matching:
    │   ├── Course = offering.subject.course
    │   ├── Year = offering.year
    │   └── Section = offering.section
    └── System enrolls matching students in subject
Step 3: Student can view enrolled subjects in dashboard
